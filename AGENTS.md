# Dotfiles — Agent Instructions

## Repository Structure

```
~/dotfiles/                         ← shared across machines (git repo)
├── Brewfile                        ← Homebrew packages (auto-synced by sysupdate)
├── install.sh                      ← symlink installer (safe to re-run)
├── zsh/
│   ├── env.zsh                     ← env vars, loaded by ALL zsh invocations
│   ├── rc.zsh                      ← interactive shell config
│   ├── profile.zsh                 ← login shell config (PATH, fnm, vscode)
│   ├── login.zsh                   ← post-login (compdump compilation)
│   └── functions/
│       ├── flushdns                ← flush DNS cache
│       └── sysupdate               ← system update (brew, fnm, gh)
└── config/
    ├── bat/config                  ← bat options (theme set via BAT_THEME env var)
    ├── fd/ignore                   ← shared ignore file for fd AND rg
    ├── ghostty/config              ← terminal emulator config
    ├── git/config                  ← shared git config
    ├── starship/starship.toml      ← prompt config
    └── tmux/tmux.conf              ← tmux config

~/dotfiles.local/                   ← machine-specific (NOT versioned)
├── config/
│   └── git/config                  ← user identity, credentials, maintenance repos
└── zsh/
    ├── env.zsh                     ← secrets, credentials, machine-specific env vars
    ├── profile.zsh                 ← machine-specific PATH entries
    ├── rc.zsh                      ← machine-specific aliases, functions
    └── functions/                  ← machine-specific autoloaded functions
```

## Zsh Load Order

1. `.zshenv` → `zsh/env.zsh` → `$DOTFILES_LOCAL/zsh/env.zsh`
2. `.zprofile` → `zsh/profile.zsh` → `$DOTFILES_LOCAL/zsh/profile.zsh`
3. `.zshrc` → `zsh/rc.zsh` → `$DOTFILES_LOCAL/zsh/rc.zsh`
4. `.zlogin` → `zsh/login.zsh` (background compdump compilation)

## Key Patterns

### Local Overrides via `~/.dotfiles.local/`

All machine-specific configuration goes in `~/dotfiles.local/`. This directory is NOT versioned. The shared dotfiles source from it using the `$DOTFILES_LOCAL` variable (set in `.zshenv`).

- **Secrets & credentials** → `zsh/env.zsh`
- **Machine-specific PATHs** → `zsh/profile.zsh`
- **Machine-specific aliases/functions** → `zsh/rc.zsh`
- **Git identity & credentials** → `gitconfig` (included at the end of shared gitconfig via `[include]`)

The gitconfig `[include]` is at the END of the shared gitconfig so local values override shared ones.

### Caching Strategy

Slow tool inits are cached in `~/.cache/` and regenerated when the tool binary changes:

| Cache file | Generated by | Invalidated when |
|---|---|---|
| `~/.cache/brew-shellenv.zsh` | `brew shellenv` | brew binary changes |
| `~/.cache/zoxide-init.zsh` | `zoxide init zsh` | zoxide binary changes |
| `~/.cache/starship-init.zsh` | `starship init zsh` | starship binary changes |
| `~/.cache/vscode-shell-integration.zsh` | `code --locate-shell-integration-path` | file doesn't exist |
| `~/.cache/zcompdump` | `compinit` | managed by `.zlogin` |
| `~/.cache/zsh_history` | zsh | — |
| `~/.cache/zsh_sessions/` | macOS session restore | — |

Pattern for caching:
```zsh
_cache="$HOME/.cache/tool-init.zsh"
if [[ ! -f "$_cache" || "$_cache" -ot "$(command -v tool)" ]]; then
  tool init zsh > "$_cache"
fi
source "$_cache"
```

### Light/Dark Mode

macOS appearance is detected automatically via `defaults read -g AppleInterfaceStyle`. A `precmd` hook (`update-appearance`) checks before every prompt and sets:

- `BAT_THEME` → `"Catppuccin Latte"` or `"Catppuccin Mocha"`
- Ghostty handles this natively: `theme = light:"Catppuccin Latte",dark:"Catppuccin Mocha"`

### Ignore Patterns (fd/rg)

A single ignore file at `config/fd/ignore` is symlinked to both:
- `~/.config/fd/ignore` (for `fd`)
- `~/.rgignore` (for `rg`)

Add patterns here to exclude from fzf search (Ctrl+T, Alt+C) and ripgrep.

### Completion System

Completion uses `compinit` + **fzf-tab**. Tab opens fzf when there are multiple matches, with previews (files via `bat`, directories via `eza`). Case-insensitive matching is enabled.

History search: ↑/↓ arrows do prefix-filtered search, Ctrl+R opens fzf fuzzy search.

### Ghostty

Ghostty is the terminal emulator. Key features configured:
- Catppuccin theme with automatic light/dark switching
- `macos-option-as-alt` for Alt keybinds (fzf Alt+C, etc.)
- Shell integration with sudo wrapper and SSH terminfo
- Quick terminal via `Cmd+`` (requires Accessibility permissions)

Tmux integration is available but not enabled by default. To enable, add `command = /opt/homebrew/bin/tmux new-session -A -s main` to `config/ghostty/config`.

### Git Diff/Merge Tools

- `git difftool` — opens VS Code by default (`-t webstorm` for WebStorm)
- `git mergetool` — VS Code or WebStorm available
- `prompt = false` — auto-opens next file without confirmation
- WebStorm handles new files gracefully (falls back to single-file view)

### `dot` CLI

The `dot` command is an autoloaded zsh function for managing dotfiles. Available commands:

| Command | What it does |
|---|---|
| `dot update` | Brew upgrade, sync Brewfile, update fnm + gh extensions |
| `dot edit` | Open dotfiles in `$EDITOR` |
| `dot doctor` | Check symlinks, tools, directories, env vars |
| `dot clean` | Clear all caches (brew, npm, bat, zsh init caches) |
| `dot link` | Re-run symlinks via Makefile |
| `dot brew` | Install Homebrew packages from Brewfile |
| `dot reload` | Reload shell (`exec zsh`) |
| `dot cd` | cd into dotfiles directory |

`dot` is a function (not a script in `bin/`) because `dot cd`, `dot reload`, and `dot update` need to modify the current shell state.

## How to Make Changes

### Adding a new shared config file
1. Create the file under `~/dotfiles/config/<app>/`
2. Add a `ln` line to `Makefile` under the `link` target
3. Run `make link` to create the symlink

### Adding a new alias or function (shared)
- **Alias** → add to `zsh/rc.zsh` under ALIASES section
- **Function** → create file in `zsh/functions/`, add `autoload -Uz funcname` in rc.zsh

### Adding machine-specific config
- Edit the appropriate file in `~/dotfiles.local/` — never edit the shared dotfiles for machine-specific values

### Machine identity
- `$MACHINE` is set to `personal` by default (in `env.zsh`)
- Override in `dotfiles.local/zsh/env.zsh` (e.g. `export MACHINE="work"`)
- Use `[[ "$MACHINE" == "work" ]]` for conditional config

### OS detection
- `$IS_MACOS` / `$IS_LINUX` booleans set in `env.zsh`

### Updating system
- Run `dot update` — updates brew, dumps Brewfile, updates fnm and gh extensions

### Adding entries to PATH
- **Shared PATH** → `zsh/profile.zsh` (login shell, runs once)
- **Machine-specific PATH** → `~/dotfiles.local/zsh/profile.zsh`
- **Never** put PATH exports in `rc.zsh` (runs per interactive shell, causes duplication)

### Setting up on a new machine
```bash
git clone <repo> ~/dotfiles
cd ~/dotfiles && make install
# Create machine-local overrides:
mkdir -p ~/dotfiles.local/zsh/functions ~/dotfiles.local/config/git
echo 'export MACHINE="work"' > ~/dotfiles.local/zsh/env.zsh
```
